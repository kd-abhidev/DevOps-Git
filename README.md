# Git Notes

## Overview

Git is a distributed version control system created by Linus Torvalds in 2005. Git tracks changes in source code and maintains a complete history of project updates. Developers use Git to manage code, work on different features, and maintain stable versions of applications.

Git supports distributed development. Each developer holds a local copy of the repository with full project history. This structure allows fast operations and offline development. Changes move to remote repositories when collaboration or sharing becomes necessary.

Platforms such as GitHub provide remote repository hosting. These platforms support collaboration between developers, project management, and version tracking for software projects.


## Purpose of This Repository

This repository contains structured notes related to Git concepts. The notes cover topics such as version control basics, Git architecture, branching, merging, and change management. The content supports learning, revision, and interview preparation for developers and DevOps learners.
